# 8.6 HR Metrics Dashboard

## 8.6.1 Business Purpose

Provide a consolidated, real‑time visual interface for HR leaders and executives to monitor key workforce metrics, support data‑driven decision making, and communicate performance across the organization.

## 8.6.2 Business Scope

- **Metric Consolidation** – Aggregate data from workforce planning, headcount, talent analytics, diversity, engagement, and compensation modules.
- **Dashboard Design** – Define layout, visualizations, drill‑down capabilities, and role‑based access.
- **Data Refresh** – Implement near‑real‑time data pipelines and scheduled refresh cycles.
- **Alerting** – Configure threshold‑based alerts for critical KPI deviations.
- **Self‑Service** – Enable managers to create ad‑hoc views and export data.
- **Governance** – Establish ownership, review cycles, and compliance checks for dashboard content.

## 8.6.3 Dashboard Schema

```sql
Table: hr_dashboard
Fields:
- id: UUID (Primary Key)
- dashboard_name: VARCHAR(200) (Required)
- description: TEXT
- layout_json: JSON (Defines widget arrangement)
- filters_json: JSON (Default filter set)
- owner_user_id: UUID (Foreign Key to user)
- is_active: BOOLEAN (Default: TRUE)
- created_at: TIMESTAMP (Auto)
- updated_at: TIMESTAMP (Auto)

Indexes:
- PRIMARY KEY (id)
- UNIQUE KEY uk_dashboard_name (dashboard_name)
- INDEX idx_owner (owner_user_id)
```

## 8.6.4 Widget Definition Schema

```sql
Table: dashboard_widget
Fields:
- id: UUID (Primary Key)
- dashboard_id: UUID (Foreign Key to hr_dashboard)
- widget_type: ENUM('KPI','Chart','Table','Map','Text')
- data_source: VARCHAR(200) (e.g., view name, query)
- config_json: JSON (Chart settings, thresholds)
- position_order: INTEGER (Render order)
- created_at: TIMESTAMP (Auto)
- updated_at: TIMESTAMP (Auto)

Indexes:
- PRIMARY KEY (id)
- INDEX idx_dashboard (dashboard_id)
- INDEX idx_widget_type (widget_type)
```

## 8.6.5 Data Source Views

```sql
View: vw_hr_overview AS
SELECT
  d.dashboard_name,
  w.widget_type,
  w.data_source,
  w.config_json
FROM hr_dashboard d
JOIN dashboard_widget w ON d.id = w.dashboard_id
WHERE d.is_active = TRUE;
```

## 8.6.6 Access Control Schema

```sql
Table: dashboard_access
Fields:
- id: UUID (Primary Key)
- dashboard_id: UUID (Foreign Key to hr_dashboard)
- user_id: UUID (Foreign Key to user)
- access_level: ENUM('View','Edit','Admin')
- granted_at: TIMESTAMP (Auto)
- expires_at: DATE (nullable)

Indexes:
- PRIMARY KEY (id)
- UNIQUE KEY uk_dashboard_user (dashboard_id, user_id)
- INDEX idx_access_level (access_level)
```

## 8.6.7 Workflow & Approvals

1. **Requirement Gathering** – HR leadership defines required KPIs and visualizations.
2. **Design** – UX team creates mockups; data architects map source views.
3. **Development** – Build dashboard definition and widgets; configure data pipelines.
4. **Review** – Dashboard owner validates data accuracy and visual integrity.
5. **Approval** – Senior HR executive signs off; access permissions are assigned.
6. **Publish** – Dashboard goes live; alerts are activated.
7. **Monitoring** – Data quality team monitors refresh failures; governance reviews quarterly.

## 8.6.8 KPIs & Metrics

| KPI | Definition | Target |
|-----|------------|--------|
| **Dashboard Availability** | % of time dashboard is accessible without errors | ≥ 99.5 % |
| **Data Freshness** | Average lag between source system update and dashboard refresh | ≤ 15 minutes |
| **User Adoption** | % of target audience regularly accessing the dashboard | ≥ 80 % |
| **Alert Response Time** | Avg. time to acknowledge and act on KPI alerts | ≤ 4 hours |
| **Change Request Cycle** | Avg. time to implement approved dashboard enhancements | ≤ 10 business days |

## 8.6.9 Best Practices

- **Standardized KPIs** – Use organization‑wide definitions to ensure consistency.
- **Role‑Based Views** – Tailor widget sets to executive, manager, and analyst audiences.
- **Performance Optimization** – Cache heavy queries; use incremental loads where possible.
- **Documentation** – Maintain a data dictionary for each widget’s source and calculation.
- **Continuous Feedback** – Collect user suggestions quarterly and prioritize improvements.

## 8.6.10 Governance & Compliance

- **Owner**: HR Analytics Lead (user reference)
- **Review Cycle**: Quarterly
- **Compliance**: Ensure all displayed data respects GDPR/CCPA privacy rules; mask PII where required.
- **Audit Log**: Record changes to dashboard definitions, widget configurations, and access grants.

## 8.6.11 Future Enhancements

- **AI‑Driven Insights** – Auto‑suggest actionable recommendations based on KPI trends.
- **Natural Language Query** – Enable users to ask questions like “show turnover by department last quarter”.
- **Mobile Optimisation** – Responsive design for on‑the‑go access via smartphones.
- **Embedded Analytics** – Integrate key widgets into other HR applications (e.g., talent portal).
- **Predictive Alerts** – Use machine learning to forecast KPI breaches before they occur.
